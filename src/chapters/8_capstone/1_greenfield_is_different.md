Tutorials, online courses, and YouTube videos used to be the primary way developers learned a new technology. They still work, but they're passive. You watch someone else build, then try to recreate what they did. The project is theirs, not yours. The decisions are pre-made.

AI coding tools change this. You can learn by building *your* project, in a stack you've never touched, with a collaborator that explains what's happening as you go. You don't need a tutorial for the specific app you want to build. You bring the idea, the AI brings the knowledge of the stack, and you learn by doing the real thing.

That's what this portfolio project is.

## What's different about greenfield

On Rare, you joined a codebase that already had conventions, patterns, and domain rules baked in. Your job was to discover them, document them, and follow them.

On a greenfield project, you set the conventions. Your `CLAUDE.md` starts blank. The first architectural decision you make becomes the pattern everything else follows. That's more freedom and more responsibility.

On Rare, you used Claude Code on a stack you were likely already familiar with. On a new project in a new stack, the dynamic shifts. Claude becomes teacher and collaborator at the same time. Your Dev Profile in `~/.claude/CLAUDE.md` calibrates how much teaching you get. If Vue is listed under "Currently Learning," `/pair-program` will explain Vue concepts thoroughly. If React is under "Strong Skills," it won't waste your time on React basics that you already know.

## Scope still matters

Even without a deadline, scoping matters. It's easy to plan a project with ten features and never finish any of them. Define what "done" looks like before you start building. The Tandem roadmap skill helps here. It breaks work into ordered steps so you can ship incrementally and always have something that works.
